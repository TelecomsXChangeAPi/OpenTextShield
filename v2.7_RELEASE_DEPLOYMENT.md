## Quick Start (Single Container)

### 1. Pull and Run
```bash
# ARM64/Apple Silicon (Recommended for M1/M2/M3/M4 Macs)
docker run -d --name opentextshield \
  -p 8002:8002 \
  -p 8080:8080 \
  telecomsxchange/opentextshield:v2.7

# OR for x86-64 servers
docker run -d --name opentextshield \
  -p 8002:8002 \
  -p 8080:8080 \
  telecomsxchange/opentextshield:v2.7-amd64
```

### 2. Wait for Startup (~2-3 minutes)
```bash
# Watch logs
docker logs -f opentextshield

# Wait for: "INFO: Uvicorn running on http://0.0.0.0:8002"
```

### 3. Verify Health
```bash
curl http://localhost:8002/health | jq .

# Expected response:
# {
#   "status": "healthy",
#   "version": "2.6.0",
#   "models_loaded": {
#     "mbert_multilingual": true
#   },
#   "timestamp": "2025-10-23T20:51:08.162237+00:00"
# }
```

### 4. Test API
```bash
curl -X POST http://localhost:8002/predict/ \
  -H "Content-Type: application/json" \
  -d '{"text":"Click here to win free money","model":"ots-mbert"}'

# Expected response:
# {
#   "label": "spam",
#   "probability": 0.9996626377105713,
#   "processing_time": 0.4347236156463623,
#   "model_info": {
#     "name": "OTS_mBERT",
#     "version": "2.5",
#     "author": "TelecomsXChange (TCXC)",
#     "last_training": "2024-03-20"
#   }
# }
```

### 5. Access Frontend
- **Frontend**: http://localhost:8080
- **API Docs**: http://localhost:8002/docs
- **OpenAPI Spec**: http://localhost:8002/openapi.json

## Production Deployment (10-Server Load Balanced)

### 1. Deploy with Docker Compose
```bash
# Start all 10 API servers + nginx load balancer
docker compose -f docker-compose.10x.yml up -d

# Monitor startup (3-5 minutes)
docker compose -f docker-compose.10x.yml logs -f
```

### 2. Verify All Servers
```bash
# Check container status
docker compose -f docker-compose.10x.yml ps

# Expected: All containers "Up" and "(healthy)"
```

### 3. Verify Load Balancer
```bash
# Test through load balancer (port 8002)
for i in {1..5}; do
  curl -X POST http://localhost:8002/predict/ \
    -H "Content-Type: application/json" \
    -d '{"text":"test","model":"ots-mbert"}' | jq .processing_time
done

# Expected: ~70-90ms latency per request
```

### 4. Monitor Performance
```bash
# Real-time stats
docker compose -f docker-compose.10x.yml stats

# Individual server health
for i in {1..10}; do
  curl -s http://localhost:900$i/health | jq .status
done
```

## Kubernetes Deployment

### 1. Create Namespace
```bash
kubectl create namespace opentextshield
```

### 2. Pull Secrets
```bash
kubectl create secret docker-registry regcred \
  --docker-server=docker.io \
  --docker-username=YOUR_DOCKER_USERNAME \
  --docker-password=YOUR_DOCKER_PASSWORD \
  -n opentextshield
```

### 3. Deploy Service & Ingress
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: opentextshield
  namespace: opentextshield
spec:
  replicas: 5  # Or more for production
  selector:
    matchLabels:
      app: opentextshield
  template:
    metadata:
      labels:
        app: opentextshield
    spec:
      imagePullSecrets:
      - name: regcred
      containers:
      - name: api
        # Use v2.7-amd64 for x86-64 Kubernetes clusters
        # Use v2.7 for ARM-based Kubernetes (k3s on Pi, AWS Graviton, etc.)
        image: telecomsxchange/opentextshield:v2.7-amd64
        ports:
        - containerPort: 8002
        livenessProbe:
          httpGet:
            path: /health
            port: 8002
          initialDelaySeconds: 60
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8002
          initialDelaySeconds: 30
          periodSeconds: 5
        resources:
          requests:
            memory: "2Gi"
            cpu: "1000m"
          limits:
            memory: "4Gi"
            cpu: "2000m"
---
apiVersion: v1
kind: Service
metadata:
  name: opentextshield
  namespace: opentextshield
spec:
  selector:
    app: opentextshield
  ports:
  - port: 80
    targetPort: 8002
  type: LoadBalancer
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: opentextshield
  namespace: opentextshield
spec:
  rules:
  - host: api.opentextshield.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: opentextshield
            port:
              number: 80
```

### 4. Deploy
```bash
kubectl apply -f deployment.yaml

# Monitor
kubectl -n opentextshield get pods -w
kubectl -n opentextshield logs -f deployment/opentextshield
```

## AWS EC2 Deployment

### 1. Launch Instance
```bash
# Use Ubuntu 24.04 LTS (t3.xlarge or larger)
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.xlarge \
  --key-name your-key \
  --security-groups default
```

### 2. Install Docker
```bash
ssh -i your-key.pem ubuntu@your-instance-ip

# Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Add to group
sudo usermod -aG docker $USER
newgrp docker

# Verify
docker run hello-world
```

### 3. Deploy
```bash
# Clone or download docker-compose file
curl -O https://raw.githubusercontent.com/TelecomsXChangeAPi/OpenTextShield/main/docker-compose.10x.yml

# Start deployment
docker compose -f docker-compose.10x.yml up -d

# Monitor
docker compose logs -f
```

### 4. Configure Load Balancer
```bash
# AWS Application Load Balancer
aws elbv2 create-load-balancer \
  --name opentextshield-alb \
  --subnets subnet-xxxxx subnet-yyyyy \
  --security-groups sg-xxxxx \
  --scheme internet-facing \
  --type application

# Target group pointing to EC2 instance port 8002
aws elbv2 create-target-group \
  --name opentextshield-targets \
  --protocol HTTP \
  --port 8002 \
  --vpc-id vpc-xxxxx

# Register targets
aws elbv2 register-targets \
  --target-group-arn arn:aws:... \
  --targets Id=i-xxxxx,Port=8002
```

## Troubleshooting

### Container Won't Start
```bash
# Check logs
docker logs opentextshield

# Common issues:
# - Port already in use: docker ps | grep 8002
# - Insufficient memory: docker stats
# - Disk space: df -h

# Solutions:
docker stop opentextshield
docker rm opentextshield
docker system prune -f
docker run -d --name opentextshield ... # retry
```

### Slow Responses (>1 second)
```bash
# Check if model is loaded
docker exec opentextshield curl -s http://localhost:8002/health | jq .

# Check container resources
docker stats opentextshield

# If CPU maxed out: reduce concurrency or add more servers
# If memory maxed out: increase container memory limit
```

### API Returns 502 Bad Gateway
```bash
# Nginx is up but backend is down. Wait for startup:
sleep 30

# Check individual servers (if using compose)
docker compose ps

# Restart if needed
docker compose restart
```

### High Memory Usage
```bash
# Model takes ~2GB per container
# 10 servers × 2GB = 20GB RAM needed

# For limited resources, use smaller deployment:
docker compose -f docker-compose.2x.yml up -d
```

## Upgrading to v2.7

### From v2.6 or Earlier
```bash
# Backup data if using persistent volumes
docker volume ls | grep opentextshield

# Stop old version
docker compose down

# Pull new version
docker pull telecomsxchange/opentextshield:v2.7

# Update docker-compose.yml to use v2.7
# Then restart
docker compose up -d

# Verify
curl http://localhost:8002/health
```

### Rolling Update (Zero Downtime)
```bash
# For Kubernetes deployments
kubectl set image deployment/opentextshield \
  opentextshield=telecomsxchange/opentextshield:v2.7 \
  -n opentextshield

# For Docker Compose with load balancer
docker compose up -d --scale api=20  # Increase replicas
# Wait for new servers to health check
docker compose up -d --scale api=10  # Remove old servers
```

## Security Considerations

### Network Security
```bash
# Only expose through load balancer
- Don't expose individual API containers
- Use VPC/security groups to restrict access
- Enable SSL/TLS at load balancer level

# Example with nginx reverse proxy:
```

### API Security
```bash
# Rate limiting (via nginx)
limit_req_zone $binary_remote_addr zone=api:10m rate=100r/s;
limit_req zone=api burst=200 nodelay;

# IP whitelist
allow 203.0.113.0/24;  # Your office
deny all;
```

### Container Security
```bash
# Run as non-root (default in v2.7)
docker run --user 1000:1000 telecomsxchange/opentextshield:v2.7

# Use read-only filesystem
docker run --read-only --tmpfs /tmp telecomsxchange/opentextshield:v2.7

# Drop capabilities
docker run --cap-drop=ALL --cap-add=NET_BIND_SERVICE telecomsxchange/opentextshield:v2.7
```

```
Hardware: M1 MacBook Pro

- Inference latency: ~400ms (first), ~70ms (subsequent)

```
Hardware: 8-core server with 32GB RAM
- Startup time: 3-5 minutes
- Load balancer latency: <10ms
- Throughput: 172 req/s (300 concurrent)
- Utilization: ~60-70% CPU, ~70% memory

### Scaling Guidelines
```
CPU-bound workload, model size ~2GB per container:

1 server:    ~10 req/s, 2.5GB RAM
5 servers:   ~50 req/s, 12.5GB RAM
10 servers:  ~100 req/s, 25GB RAM
20 servers:  ~200 req/s, 50GB RAM

Add more servers to increase throughput linearly
```

## Incident Response

### API Degradation
1. Check health: `curl http://localhost:8002/health`
2. Check resources: `docker stats`
3. Check logs: `docker logs opentextshield`
4. Restart container: `docker restart opentextshield`

### Memory Leak
1. Monitor memory: `docker stats --no-stream opentextshield`
2. If increasing over time: restart container
3. Check model loading: `docker exec opentextshield python -c "import torch; print(torch.cuda.memory_allocated())"`

### Load Balancer Issues
1. Check backend servers: `docker compose ps`
2. Check nginx logs: `docker compose logs load-balancer`
3. Verify port 8002: `netstat -tulpn | grep 8002`
